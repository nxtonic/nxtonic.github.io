cool things that I've worked on

specs

Specs is something I came up with after spending a lot of time around my peers. Generally, they learn a lot by following tutorials, but always had a kind word to say about my random, made-up experiments with different technologies.

The site is essentially a collection of experimental ideas for projects that vary in scale and difficulty, but also have very few guidelines. This is to foster creative problem solving, away from a more spoon-fed approach.

I've always felt that building functioning products from scratch using your imagination is a really great way to build up your skills, and know everything about the codebase you're working with.

idea-factory

Forgive me for getting "meta" for a moment, but I came up with this idea to generate entrepreneurially viable ideas.

I also wanted to see how performant I could make a web application without using any external libraries or frameworks. The result is pretty good, with it loading from github.io in a time of ~250ms if you have a good connection. I'm pretty happy with that.

This was also good practice at dealing with the HTML5 history api and a few other little javascript tidbits that I'd never messed about with before. It's still in progress, but has been good fun so far.

At some point this project will be scaled up and include the submission of new ideas and possibly even automatically generated ideas based on machine-learning.

magic8

I never had a magic 8 ball as a kid, so I decided to make myself one. You can use it too if you'd like, it's pretty (in)accurate.

Development only started on this in April 2016, so it's pretty stock-standard so far. I'm planning to practice my SVG drawing and animation skills as well as employing javascript best practices to create a framework-less application.
